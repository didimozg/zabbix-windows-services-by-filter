# Security Policy

## Supported Versions

Security fixes are applied to the latest commit on the default branch.
Older commits, branches, and tags may not receive fixes.

## Reporting A Vulnerability

If you believe you found a security issue:

1. Do not open a public issue for sensitive vulnerabilities.
2. Use GitHub private vulnerability reporting if it is enabled for this repository.
3. If private reporting is unavailable, contact the maintainer through the GitHub profile and include clear technical details.

Please include:

- affected repository, branch, tag, or commit
- impact and attack scenario
- reproduction steps or proof of concept
- affected files, scripts, or configuration
- suggested mitigation or workaround, if known

Non-sensitive hardening ideas can still be opened as normal issues.
